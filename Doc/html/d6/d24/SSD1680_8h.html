◆ SSD1680_Color

enum SSD1680_Color

Defines color.

Note
ColorRed and ColorAnotherRed take precedence over ColorBlack and ColorWhite 'cause secondary color channel is being applied after primary color image is formed. No color blending can be performed. I.e. if pixel is "red" then it doesn't matter what color is underneath and if pixel is not "red" then primary color takes effect.

◆ SSD1680_ScanMode

enum SSD1680_ScanMode

Defines source (column) scanning mode.

Choose the value depending on your particular display resolution. Smaller displays (i.e. 152x152) uses narrow scan and have first and last 8 columns addressable but out of visible field.

Note
Using wrong value may cause image to be offset by 8 pixels or even distorted.
Enumerator

Full width 0 to 175 column scan

NarrowScan 

Narrow width 8 to 167 column scan

enum SSD1680_RAMBank

Defines RAM bank either for primary (black) or secondary (usually red) color.

Using RAMRed with monochrome display does either nothing or shades of gray on some models.

Note
Don't be confused by the name. RAMRed is also applicable to any secondary color whatever it is on your particular display.

◆ SSD1680_DataEntryMode

enum SSD1680_DataEntryMode

Address counter increment behavior.

SSD1680 has 3 flags defining the directions of memory counter increment/decrement on bulk data transfer. Depending on these flags sequential bytes are put into RAM in different order.

RightThenDown is suitable for most cases.

See also
https://v4.cecdn.yun300.cn/100001_1909185147/SSD1680.pdf page 23
Enumerator
LeftThenUp 

X decrements then Y decrements. Starts from bottom-right corner and goes left.

RightThenUp 

X increments then Y decrements. Starts from bottom-left corner and goes right.

LeftThenDown 

X decrements then Y increments. Starts from top-right corner and goes left.

RightThenDown 

X increments then Y increments. Starts from top-left corner and goes right.

UpThenLeft 

Y decrements then X decrements. Starts from bottom-right corner and goes up.

UpThenRight 

Y decrements then X increments. Starts from bottom-left corner and goes up.

DownThenLeft 

Y increments then X decrements. Starts from top-right corner and goes down.

DownThenRight 

Y increments then X increments. Starts from top-left corner and goes gown.
